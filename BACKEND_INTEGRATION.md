
# 🚀 Инструкция по соединению Фронтенда (GigaQifa) и Бэкенда (Python)

Этот документ — ваш путеводитель по «оживлению» интерфейса. Здесь написано, как передать данные из ваших Python-скриптов прямо в это приложение.

---

## 🧩 Общая логика: как это работает?
Фронтенд (это приложение) — это красивая оболочка. Бэкенд (ваш Python сервер на FastAPI) — это «мозги». 
Чтобы они работали вместе, Фронтенд делает **запросы** (спрашивает данные), а Бэкенд присылает **ответы** (в формате JSON — это просто структурированный текст).

---

## 📦 Шаг 1: Настройка доступа (CORS)
Ваш Python-сервер должен разрешить фронтенду забирать данные. В файле `main.py` на бэкенде обязательно добавьте этот блок:

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# РАЗРЕШАЕМ ДОСТУП
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"], # Позже здесь будет адрес вашего VPS
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 📊 Шаг 2: Обновление счетчиков (Экран "Поиск Лидов")
Когда вы открываете вкладку «Поиск Лидов», фронтенд обращается по адресу: `GET /api/v1/leads/stats`.

### Что должен прислать Python (Формат данных):
Ваш скрипт должен вернуть такой "список" цифр:
```json
{
  "totalInWork": 5000,      // Контакты в работе (общий итог)
  "totalProcessed": 500,    // Обработанные контакты (общий итог)
  "totalWarmLeads": 42,     // Тёплые лиды (общий итог)
  "groups": [
    { "id": "telegram", "processed": 0, "inWork": 0, "warmLeads": 0 },
    { "id": "vk", "processed": 0, "inWork": 0, "warmLeads": 0 },
    { "id": "max", "processed": 0, "inWork": 0, "warmLeads": 0 }
  ]
}
```

### Пример функции на Python:
```python
@app.get("/api/v1/leads/stats")
async def get_stats():
    # Здесь ваша логика подсчета из базы данных или файлов
    return {
        "totalInWork": 5000,
        "totalProcessed": 500,
        "totalWarmLeads": 42,
        "groups": [
            {"id": "telegram", "processed": 0, "inWork": 0, "warmLeads": 0},
            {"id": "vk", "processed": 0, "inWork": 0, "warmLeads": 0},
            {"id": "max", "processed": 0, "inWork": 0, "warmLeads": 0}
        ]
    }
```

## 💬 Шаг 3: Работа Чата (Общение с ИИ)
Когда пользователь пишет сообщение в чате, фронтенд отправляет запрос на `POST /orchestrator/process`.

### Что получает Python:
```json
{
  "query": "Привет, проанализируй этот файл",
  "contextFiles": ["uuid-файла-123"] # Список ID файлов, если они были прикреплены
}
```

### Что Python должен ответить:
```json
{
  "answer": "Конечно! Я изучил данные и пришел к выводу...",
  "confidence": 0.98,
  "sources": ["отчет_январь.pdf"]
}
```

## 📎 Шаг 4: Загрузка файлов (Скрепка)
Когда нажимают на скрепку и выбирают файл, он летит на `POST /api/v1/files/upload`.

**Задача бэкенда:**
1. Принять файл.
2. Сохранить его (в папку или облако).
3. Вернуть уникальный ID этого файла (например, `abc-123`), чтобы фронтенд мог отправить этот ID в чат для контекста.

## 🚀 Как запустить всё это на VPS?

1. **Фронтенд**: 
   - Выполните `npm run build`. 
   - Папку `.next/standalone` перенесите на сервер.
   - Запустите через `node server.js` (рекомендуется использовать `pm2`).

2. **Бэкенд (Python)**:
   - Установите библиотеки: `pip install fastapi uvicorn`.
   - Запустите сервер: `uvicorn main:app --host 0.0.0.0 --port 8000`.

3. **Связь**:
   - В настройках фронтенда (файл `.env`) укажите `NEXT_PUBLIC_API_URL=http://IP_ВАШЕГО_СЕРВЕРА:8000/api/v1`.
